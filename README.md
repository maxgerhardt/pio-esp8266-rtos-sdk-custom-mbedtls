# pio-esp8266-rtos-sdk-custom-mbedtls
Contains an example for building a customized mbedtls library with the esp8266-rtos-sdk framework.

### Attention

For building this, you **must** temporarily rename a folder in PIO's framework files. PIO automatically sets the include flag ` -I $PIOHOME_DIR\packages\framework-esp8266-rtos-sdk\include`, in which there is a "mbedtls" folder. This folder contains some outdated defines and conflicts with a custom library. 

For sucessful building, you must go to your PIO home folder (Windows: `C:\Users\<Username>\.platformio`, Linux/Mac: `/home/<user>/.platformio`), navigate to `packages/framework-esp8266-rtos-sdk/include` and temporarily rename the `mbedtls` folder to e.g. `mbedtls-2`.

The build produces a `libmedtls-custom.a` library which you can use in further builds once the correct settings were applied. You can then link against this static library.
